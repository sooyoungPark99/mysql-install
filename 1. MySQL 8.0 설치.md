# MySQL 8.0 설치

Oracle Linux 7.9 환경에서 MySQL 8.0 Community Server를 설치한다.

---

### 1. 실습 환경

| 항목 | 값 |
|------|-----|
| OS | Oracle Linux 7.9, UEK 5.4.17 |
| IP | 192.168.23.67 |
| SELinux | Enforcing |
| firewalld | 활성 (기본 3306 미개방) |
| 목표 | MySQL 8.0 Community Server 설치 |

> 기존 상태: MariaDB user/group(UID/GID 27) 잔재 가능, /etc/my.cnf 존재 가능

---

### 2. 작업 계정

이 문서의 전체 작업은 root에서 수행한다.

```bash
su -
```

### 4. MariaDB 패키지 제거

```bash
yum remove mariadb-libs mariadb mariadb-server -y
```

설정 파일 잔여 여부를 확인하고 백업한다.

```bash
ls -la /etc/my.cnf /etc/my.cnf.d/ 2>/dev/null

[ -f /etc/my.cnf ] && cp /etc/my.cnf /etc/my.cnf.mariadb.bak || echo "/etc/my.cnf 없음"
```

> `&&`: 앞 명령 성공 시 뒤 명령 실행
> `||`: 앞 명령 실패 시 뒤 명령 실행

### 9. 보안 초기화 (mysql_secure_installation)

```bash
mysql_secure_installation
```

| 질문 | 권장 답변 |
|------|---------|
| New password | 새 비밀번호 입력 (정책 충족) |
| Disallow root login remotely? | Y (실습 목적이면 N 가능) |
| Reload privilege tables? | Y |

#### root 접속을 테스트한다.

```bash
mysql -u root -p
```

### 10. firewalld 포트 개방 (3306)

외부 접속이 필요하면 3306/tcp를 개방한다.

```bash

firewall-cmd --reload
firewall-cmd --list-all
```

### 13. 트러블슈팅 - Access denied로 초기화 필요할 때

skip-grant-tables 모드로 기동한다.

```bash
systemctl stop mysqld
mysqld --skip-grant-tables --skip-networking --user=mysql &
sleep 3
mysql -u root
```

비밀번호 정책을 낮추고 비밀번호를 변경한다.

```sql
flush privileges;
set global validate_password.policy = low;
set global validate_password.length = 6;
alter user 'root'@'localhost' identified by 'oracle';
exit;
```

정상 모드로 재기동한다.

```bash
kill $(pgrep -f skip-grant-tables)
systemctl start mysqld
mysql -u root -poracle
```

---

### 14. 트러블슈팅 요약

| 증상 | 원인 | 해결 |
|------|------|------|
| Access denied (root@localhost) | 임시 비밀번호 오류 / 초기화 필요 | 13번 단계 수행 |
| 서비스 시작 실패 (SELinux) | 비표준 경로 컨텍스트 미설정 | 11번 단계의 fcontext / restorecon 적용 |
